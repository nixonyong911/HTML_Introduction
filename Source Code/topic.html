About XML Http Request

History

The first appearance of XMLHttpRequest was in Internet Explorer 5 which was released by Microsoft and is known to be the key behind most of the key fundamental technology of many modern web application; Google Email. Although it contains XML in its name, it was not originally invented to be tied to XML only. Moreover, despite its revolutionary role in AJAX, the early release of XMLHttpRequest has very limited capability such as text-based-only data transfer and unable to handle cross-domain request.

Soon after, Mozilla also implemented its own version or XMLHttpRequest in contrast to Microsoft’s and many other browser such as Safari and Opera followed. In 2006, W3C has published the best practice for XMLHttpRequest and where AJAX and Web 2.0 was released, whereby after that it became widely used everywhere. In 2008, XMLHttpRequest level 2 draft was released which include few new features such as support for binary and text-based data transfer, safe cross-origin request and many more. Lastly, in 2011, XMLHttpRequest level 2 was finally merged with the original XMLHttpRequest.

Overview of XMLHttpRequest

More about XMLHttpRequest

Although XMLHttpRequest was created by Microsoft and has XML in the name, it is now been standardized by WHATWG and moved to W3C in 2006. Moreover, it can receive any kind of data, and support different type of protocols other than HTTP, and different kind of web browser. Lastly it allow JavaScript to run synchronously, however, it is not recommended to run synchronously since all other operation is halted until the current one is completed, but in some case where the current request need to be completed before able to proceed to next task.

Advantage of XMLHttpRequest

  • Can be easily implemented when user have great understanding of JavaScript
  • Lower traffic in both client and server side
  • Higher interaction means better responsive on the whole page
  • User doesn’t have to spam refresh to get updated content
  • Give user instant validation

Disadvantage of XMLHttpRequest

  • It's open source, meaning anyone can gain access to the source code
  • Due to its design architecture, user only able to gain information only from one server.
  • Complicated and complex
  • Require JavaScript to be enabled in browser

Step of creating XMLHttpRequest

  1. Determine type of web browser where the html will be running on and create the object based on it.
  2. Create function that is able to receive data from server, this could be achieved by using onReadyStateChange property which able to store the function that responsible for receiving response from server.
  3. After creating function to receive data from server, we need to know the status of the server’s response, thus readyState property is used to confirm that the response from server is complete, below shows a table which indicate the readyState value’s meaning
  4. Now we know whether the response is complete, next we need to be able to retrieve the response, therefore this is where responseText property take place. This allow user to access the server’s response.
  5. After getting all the responses from server, now we are ready to send server some request. First, in Ajax application, we need to define the URL of server-side script. Next we use the function to send the request.
  6. Lastly, after we had done writing the xmlhttprequest, we will have to implement in our html file. This can be done by using onChange attribute.
